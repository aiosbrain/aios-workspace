# scaffold/ ‚Äî this directory IS the product

Everything here gets stamped into every user's workspace by `scripts/scaffold-project.sh`
and kept current afterward by `aios update` (3-way merge via `toolkit-merge.mjs`).
**Edit the template, never a stamped copy** ‚Äî a fix in someone's stamped workspace doesn't
ship to anyone else. Blast radius: every contributor's personal repo, on their next
`aios update`.

## Three contexts ‚Äî a change must hold for all of them

`--context consultant | employee | business-owner` (`scripts/scaffold-project.sh`).
Consultant and business-owner share the same `0-context`/hours/client-surface skin;
business-owner additionally gets a sanctioned `6-business/` sibling root (bookkeeping,
entities, insurance, engagements, partnerships ‚Äî outside `sync_include`). Test all three
contexts when touching scaffold content, not just the default (`consultant`).

## Dual-maintenance pairs ‚Äî change together

- `scaffold/.claude/rules/frontmatter.md` (the scaffold's self-contained tier-vocabulary
  copy) ‚Üî `../docs/tier-vocabulary.md` (monorepo root, i.e. `aios/docs/tier-vocabulary.md`).
- Any stamped path change ‚Üî `scripts/toolkit-manifest.mjs`'s four buckets (MANAGED /
  SEED_IF_ABSENT / PERSONAL / SCAFFOLD_UNMANAGED) ‚Äî `test/toolkit-manifest-parity.test.mjs`
  enforces this in lockstep.

## Template mechanics

`.tmpl` files (`CLAUDE.md.tmpl`, `AGENTS.md.tmpl`, `RESOLVER.md.tmpl`, `README.md.tmpl`,
`aios.yaml.tmpl`, `project.yaml.tmpl`, `workspace.yaml.tmpl`, `contacts.yaml.tmpl`,
`engagement.yaml.tmpl`, `package.json.tmpl`) get `sed`-substituted by `scaffold-project.sh`
using `{{PLACEHOLDER}}` syntax ‚Äî e.g. `{{SLUG}}`, `{{OWNER}}`, `{{CONTEXT}}`,
`{{CONTEXT_TITLE}}`, `{{STAKEHOLDER_NAME}}`/`{{STAKEHOLDER_FULL}}`/`{{STAKEHOLDER_LABEL}}`,
`{{DESC}}`, `{{MEMBER_LIST}}`/`{{MEMBER_COUNT}}`, `{{CURRENCY}}`, `{{START_DATE}}`,
`{{OUTWARD_TIER}}`/`{{OUTWARD_DESC}}`, `{{TEAM_ID}}`, `{{BRAIN_URL}}`.

## What ships where

- `scaffold/.claude/` ‚Äî `rules/`, 16 skills under `skills/` (+ `INDEX.md` catalog,
  generated by `scripts/gen-catalog.mjs`), `rubrics/`, `commands/`, `agents/`,
  `personalities/`, `descriptors/`, `settings.json`, `integrations.json`.
- `scaffold/scripts/aios.mjs` ‚Äî a **delegating shim**, not the real CLI: it forwards
  every command to the canonical toolkit checkout (`../aios/aios-workspace` or
  `AIOS_TOOLKIT_DIR`). Never vendor the full CLI here.
- `scaffold/AGENTS.md.tmpl` + `scaffold/RESOLVER.md.tmpl` live at the scaffold **root**,
  not under `.claude/` ‚Äî confirmed by directory listing, don't move them.
- `.opencode/`, `.mcp.json`/`.mcp.example.json`, `.envrc`, `.env.example`, `bin/aios` ‚Äî
  runtime/env plumbing hydrated alongside the spine.

## Manifest lockstep

Any change to what gets stamped (new file, new destination, new placeholder) must be
reflected in `scripts/toolkit-manifest.mjs`'s bucket classification ‚Äî the parity test
fails the build otherwise.
